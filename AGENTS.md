# AGENTS.md

このリポジトリは、日本の上場企業比較レポート生成ツールのMVPです。Codexや他の自動化エージェントは、以下のルールを守って作業してください。

## セキュリティとGit管理

- APIキーをコミットしない。
- `.env` と `.env.*` をコミットしない。ただし `.env.example` は管理対象に残す。
- 講義資料PDFをコミットしない。
- 生成されたWord、PDF、Excelをコミットしない。
- SQLite DB、EDINET取得ファイル、未加工データ、大量ファイルをコミットしない。
- 出力物は `output/` 配下に置く。
- `output/reports/`, `output/charts/`, `output/raw_filings/`, `output/logs/` は生成物置き場でありGit管理しない。

## 設計ルール

- 課題条件、業種判定、プリセットはYAMLに置き、コードに直書きしない。
- 大学課題モードと汎用分析モードを混同しない。
- 大学課題モードでは、2000-04-01以降上場、上場後3年以上、同じ業種、除外業種をチェックする。
- 汎用分析モードでは、金融・医療も分析対象にできる。
- 欠損値や0除算で落とさない。算定できない値は「推定不可」として注記する。
- 投資助言表現を避ける。「買い」「売り」「投資すべき」などの推奨表現を出力しない。

## レポート品質

- 表とグラフ入りWordレポートを生成できる状態を維持する。
- 課題対応表、条件適合表、＋α分析、警告、欠損注記、参考資料を維持する。
- 講義資料PDFそのものではなく、必要な内容は `docs/course_framework.md` に要約する。

## プリセット固有ルール

- コメダHD × ドトール・日レスHDは `business_theme` では許容する。
- コメダHD × ドトール・日レスHDは `strict_jpx_industry` ではJPX業種不一致として警告する。
- 航空会社プリセットでは、日本航空の再上場注記を維持する。
- 航空会社プリセットでは、燃油費、為替、機材投資、外部環境の分析観点を維持する。

## テスト

- 変更後は `pytest` が通る状態を維持する。
- EDINET APIキーがなくても、サンプルCSVだけでテストが通る状態を維持する。
- Word生成テストが重くなりすぎる場合でも、最低限の `report_writer` スモークテストは維持する。
