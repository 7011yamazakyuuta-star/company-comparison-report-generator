# Development Plan

今後のIssue候補です。MVP初回コミット後は、以下を小さなIssueに分けて進めます。

## Issue 1: Wordレポート品質改善

表の横幅、見出し、章立て、注記、グラフ配置を改善し、提出物として読みやすい体裁にする。

## Issue 2: 実データ取得と出典管理

サンプルCSVから実データCSVへ差し替える手順を整え、出典、取得日、加工方法を記録できるようにする。

## Issue 3: EDINET API取得強化

`EDINET_API_KEY` を使った文書一覧取得、保存済み一覧の検索、CSVあり有価証券報告書の1件取得はMVPに入り始めた。次は取得ログ、リトライ、期間指定取得、取得済みデータの検索UIを強化する。

## Issue 4: XBRL/CSV解析強化

EDINETのXBRLまたはCSVから主要財務数値を抽出し、現行のサンプルCSV形式にマッピングする。

## Issue 5: JPX企業マスタ取得

JPXの上場会社情報から業種、上場日、証券コード、会社名を更新できる仕組みを検討する。

## Issue 6: カフェ・外食プリセット改善

コメダHD、ドトール・日レスHD、サンマルクHDなどの比較テーマ、注記、手入力KPIを改善する。

## Issue 7: 航空会社プリセット改善

日本航空、ANA HD、スターフライヤー、スカイマークについて、燃油費、為替、機材投資、外部環境の観点を強化する。

## Issue 8: 講義フレームワーク対応表改善

講義フレームワークとレポート章の対応表をより明確にし、提出条件の自動チェックを強化する。

## Issue 9: Streamlit UI改善

プリセット選択、警告表示、手動企業選択、レポート生成後のダウンロード導線を改善する。

## Issue 10: 汎用業種プラグイン化

業種別の分析観点、追加KPI、文章テンプレートをプラグイン的に追加できる設計を検討する。

## Issue 11: GitHub Actions / CI強化

pytestに加えて、フォーマット、静的解析、最低限のWord生成スモークテストをCIに組み込む。

## Issue 12: 実企業データによるレポート検証

実データを使ってレポートを生成し、欠損、0除算、表記揺れ、出典管理の問題を洗い出す。
